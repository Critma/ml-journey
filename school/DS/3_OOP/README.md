# Object-Oriented Programming (OOP) — DS Track

## Требования

### Упражнение 00 — Простой класс
- Файл: `first_class.py`
- Задача: создать класс `Must_Read`, который читает файл `data.csv` и печатает его содержимое
- Ограничения: импорт запрещён, имя CSV-файла можно указать прямо в коде класса
- Особенность: использовать `print()` внутри класса

### Упражнение 01 — Метод
- Файл: `first_method.py`
- Задача: перенести код из тела класса в метод `file_reader()`, заменить `print()` на `return`
- Класс: `Research`
- Особенность: методы похожи на функции — они могут что-то возвращать

### Упражнение 02 — Конструктор
- Файлы: `first_constructor.py`, `data.csv`
- Задача: использовать конструктор `__init__()` для передачи пути к файлу как аргумента
- Разрешённые импорты: `import sys`, `import os`
- Валидация: проверка структуры файла (заголовок из двух строк, разделённых запятой; данные — 0 или 1)
- Поведение: выброс исключения при некорректной структуре файла

### Упражнение 03 — Вложенный класс
- Файл: `first_nest.py`
- Задача: добавить аргумент `has_header` (по умолчанию `True`) в метод `file_reader()`
- Возвращаемое значение: список списков (например, `[[0, 1], [1, 0]]`)
- Вложенный класс `Calculations`:
  - `counts()`: количество орлов и решек
  - `fractions()`: доли в процентах
- Поведение: обработка заголовка и возврат данных без него

### Упражнение 04 — Наследование
- Файл: `first_child.py`
- Задача: перенести `data` в конструктор класса `Calculations`
- Новый класс `Analytics`, наследующийся от `Calculations`:
  - `predict_random(amount)`: предсказание случайных наблюдений
  - `predict_last()`: возврат последнего элемента данных
- Разрешённые импорты: `import sys`, `from random import randint`

### Упражнение 05 — Файл конфигурации и основная программа
- Файлы: `config.py`, `analytics.py`, `make_report.py`
- Задача: вынести параметры в `config.py`, перенести логику в `analytics.py`, создать главный скрипт `make_report.py`
- Разрешённые импорты: `import os`, `from random import randint`
- Класс `Analytics`: метод `save_file(data, file_name, 'txt')` для сохранения результатов
- Шаблон отчёта в `config.py`:
  > "В нашем эксперименте с подбрасыванием монеты из 12 наблюдений пять раз выпала решка и семь раз — орёл. Вероятности составляют 41,67% и 58,33% соответственно. Мы прогнозируем, что в следующих трёх наблюдениях решка выпадет один раз, а орёл — два раза."

### Упражнение 06 — Логирование
- Файлы: `config.py`, `analytics.py`, `make_report.py`, `analytics.log`
- Задача: добавить логирование в каждый метод каждого класса
- Разрешённые импорты: `import os`, `from random import randint`, `import logging`, `import requests` (или `urllib`), `import json`
- Формат лога: дата, время и сообщение, разделённые пробелом
- Пример: `2020-05-01 22:16:16,877 Calculating the counts of heads and tails`

## Технологии

- **Классы и объекты** - базовые понятия ООП в Python (ex00-ex06)
- **Методы класса** - функции внутри класса (ex01, ex03)
- **Конструктор `__init__()`** - инициализация экземпляров класса (ex02, ex03, ex04)
- **Вложенные классы** - класс внутри другого класса (ex03)
- **Наследование** - наследование классов (`class Analytics(Research.Calculations)`) (ex04)
- **Файлы конфигурации** - вынесение параметров в отдельные файлы (ex05)
- **Логирование** - запись отладочной информации в файл с помощью `logging` (ex06)
- **Исключения** - обработка ошибок с помощью `raise Exception` (ex02, ex03, ex04)
- **random module** - генерация случайных чисел для предсказаний (ex04, ex05, ex06)
